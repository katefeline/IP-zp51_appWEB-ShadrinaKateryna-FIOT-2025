Для чого використовуються каскадні таблиці стилів (CSS)?

CSS (Cascading Style Sheets) — це мова, яка відповідає за візуальне представлення документів, написаних мовою розмітки (найчастіше HTML). Якщо HTML — це «скелет» сторінки (структура), то CSS — це її «одяг» (зовнішній вигляд).

Основні функції CSS:

  • Розділення змісту та оформлення: Це дозволяє змінювати дизайн сайту без редагування HTML-коду кожної сторінки. Ви можете змінити один CSS-файл, і вигляд усього вебсайту оновиться.
  • Керування макетом: CSS дозволяє розміщувати елементи в потрібних місцях: створювати колонки, сітки (Grid), гнучкі контейнери (Flexbox), фіксувати меню зверху тощо.
  • Стилізація тексту та кольорів: Зміна шрифтів, розміру тексту, міжрядкових інтервалів, кольору фону та тексту. Наприклад, цей текст відображається шрифтом Segoe UI завдяки CSS.
  • Адаптивність: За допомогою медіа-запитів CSS дозволяє сайту виглядати гарно на будь-яких пристроях: від великих моніторів до смартфонів.
  • Анімація та інтерактивність: Створення ефектів при наведенні миші, плавних переходів, анімації елементів без використання JavaScript.

Які методи визначення стилів HTML-документі ви знаєте

Існує три основні методи додавання стилів (CSS) до HTML-документа. Вони відрізняються пріоритетністю виконання та зручністю підтримки коду.

  • Зовнішні стилі (External Style Sheet). Це професійний стандарт і найкращий спосіб для розробки вебсайтів. Стилі зберігаються в окремому файлі з розширенням .css, який підключається до HTML-документа, використовуючи тег <link> всередині секції <head></head>.
  • Внутрішні стилі (Internal Style Sheet). Стилі пишуться безпосередньо в HTML-файлі, але виокремлюються в спеціальний блок, використовуючи тег <style> всередині секції <head></head>.
  • Вбудовані стилі (Inline Styles). Стилі додаються безпосередньо до окремих HTML-елементів за допомогою атрибута style. Цей метод має найвищий пріоритет, але його важко підтримувати при великій кількості елементів.

У чому різниця між принципами каскадування і спадкування?.

Спадкування (Inheritance) стосується того, як елементи отримують властивості від своїх батьків (зв'язок «згори вниз» по структурі HTML).Це процес, коли дочірній елемент (child) автоматично переймає певні стилі від свого батьківського елемента (parent).

Каскадування (Cascading) стосується того, як браузер вирішує конфлікти, коли до одного й того ж елемента застосовується кілька різних правил. Це алгоритм, за яким браузер визначає, яке правило є «головнішим», якщо до одного елемента застосовано кілька суперечливих стилів.

    Три фактори, що впливають на каскадування:
  • Специфічність (Specificity): Чим більш специфічне правило, тим вищий його пріоритет. Наприклад, селектор класу (.class) має вищу специфічність, ніж селектор тегу (div).
  • Порядок визначення (Source Order): Якщо два правила мають однакову специфічність, то перевага надається тому, яке визначене пізніше в коді.
  • Важливість (Importance): Правила з позначкою !important мають найвищий пріоритет і переважають над іншими.

Як в HTML-документі оформлюються селектори тегу і клас?.

Селектори тегу використовуються для вибору всіх елементів певного типу, наприклад, div або p. Селектори класу використовуються для вибору елементів з певним атрибутом класу, наприклад, .classname. Селектори класу починаються з крапки (.), а селектори тегу просто вказують ім'я тегу.

Селектор Класу використовується, коли треба стилізувати конкретну групу елементів незалежно від їхнього тегу.

Для чого використовується ідентифікатор?

Селектор id використовується для вибору конкретного елемента за його унікальним ідентифікатором, наприклад, #header. Ідентифікатори повинні бути унікальними в межах документа.

Навести приклади оформлення текстових елементів.

HTML використовує такі елементи, як <b> та <i> для форматування виведення, наприклад жирний або курсивний текст.

Елементи форматування були розроблені для відображення спеціальних типів тексту:

- Жирний текст

- Важливий текст (виділяється жирним)

- Курсивний текст

- Важливий текст (виділяється курсивом)

- Помічений (маркований) текст

- Маленький текст

- Видалений текст

- Вставлений текст

- Підрядковий текст (нижній індекс)

- Надрядковий текст (верхній індекс)
